## Background

Sepsis is a life-threatening condition that affects millions of ICU patients each year. While randomized controlled trials (RCTs) are the gold standard for establishing causality, most sepsis RCTs have yielded negative or inconclusive results. This creates an urgent need for complementary approaches that can generate and prioritize causal hypotheses from observational data.

This project applies constraint-based causal discovery algorithms to a large real-world ICU dataset to identify causal relationships between sepsis risk factors, physiologic variables, complications, and clinical outcomes. An ensemble approach is used to improve robustness against the challenges common in healthcare data: missing values, mixed variable types, and unmeasured confounders.

---

## Objectives

- Demonstrate a proof-of-concept ensemble approach to causal discovery that is robust to real-world healthcare data challenges
- Reproduce known causal relationships between sepsis risk factors, physiology, complications, and outcomes
- Identify treatment-outcome hypotheses for further investigation using causal inference techniques

---

## Dataset

Data are sourced from [MIMIC-IV v3.1](https://physionet.org/content/mimiciv/), a large publicly available critical care database. Access requires credentialing through PhysioNet.

The study cohort consists of adult ICU patients meeting Sepsis-3 criteria. The cohort is defined using the MIMIC-IV derived `sepsis3` table and linked to ICU stay, lab, medication, and outcome data.

---

## Pipeline

### 1. Cohort Definition (`sql/sepsis_complications.sql`)
A BigQuery SQL script identifies all Sepsis-3 ICU stays and computes complication flags within defined time windows relative to sepsis onset:
- Acute kidney injury (AKI) onset within 24 hours and after 24 hours, staged by KDIGO criteria
- Mechanical ventilation onset within 24 hours and after 24 hours

### 2. Feature Engineering (`scripts/feature_engineering.py`, `scripts/build_features.py`)
Features are extracted from MIMIC-IV tables and windowed around sepsis onset time. Feature groups include:

| Group | Variables |
|---|---|
| Demographics | Age, gender, race |
| Vitals | Heart rate, blood pressure, SpO2, FiO2, temperature |
| Labs | Lactate, creatinine, BUN, sodium, potassium, bilirubin, platelets, INR, WBC, hemoglobin, pH |
| Treatments | Antibiotics given, vasopressors given |
| Outcomes (24h) | AKI onset stage, mechanical ventilation onset |
| Outcomes (post-24h) | AKI progression stage, mechanical ventilation post-24h |
| Mortality | Hospital mortality |
| Anthropometrics | BMI |

### 3. Causal Discovery (`scripts/causal_discovery_fisherz.py`, `scripts/causal_discovery_kernel.py`)
Two constraint-based algorithms are applied across multiple imputation strategies and independence tests:

**Algorithms**
- PC algorithm
- Fast Causal Inference (FCI) algorithm

**Independence Tests**
- Fisher Z (linear, parametric)
- Kernel-based Conditional Independence (KCI, nonparametric)
- Missing Value PC (MV-FisherZ, handles missingness natively)

**Imputation Strategies**
- Mean imputation
- KNN imputation with standard scaling
- Raw data with missingness indicators appended as binary variables

Background knowledge is encoded as a tiered temporal structure: demographics cannot be caused by any other variable, physiology and treatments precede 24-hour complications, which precede post-24-hour outcomes, which precede hospital mortality.

### 4. Ensemble Aggregation (`scripts/causal/ensemble_table.py`, `scripts/causal/indirect_ensemble_table.py`)
Directed edges from all algorithm runs are pooled into an ensemble table. Each edge is scored by the number of runs in which it appears, providing a robustness-weighted summary of the discovered causal graph. Indirect paths between variables are also enumerated and scored.

### 5. Regression Validation (`scripts/regression`)
Logistic regression models are used to perform directionality analysis and validate selected causal hypotheses generated by the discovery step, adjusting for relevant covariates.

## Data Access

MIMIC-IV data is not included in this repository. To reproduce the cohort:
1. Complete the required CITI training and apply for access at [PhysioNet](https://physionet.org/content/mimiciv/)
2. Run `sql/sepsis_complications.sql` in BigQuery against the MIMIC-IV 3.1 dataset
3. Download the result as `data/raw/sepsis_cohort.csv`
4. Update the MIMIC-IV local path in `scripts/build_features.py`
5. Run `python scripts/build_features.py` to build the analysis dataset
